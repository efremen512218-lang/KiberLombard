# ⚡ Быстрое исправление всех ошибок

## Главная проблема:

Ошибка 400 в `/api/quote` - предметы не найдены, потому что `assetid` не совпадают.

## Решение:

### 1. Перезапустите backend с логированием

```bash
cd backend
python main.py
```

В консоли вы увидите:
```
[QUOTE] Запрос: steam_id=..., asset_ids=[...]
[QUOTE] Загружено X предметов
[QUOTE] Найдено Y выбранных предметов
```

### 2. Проверьте что показывает

Если показывает "Найдено 0 выбранных предметов" - значит `assetid` не совпадают.

### 3. Откройте консоль браузера (F12)

Посмотрите в Network → запрос к `/api/quote` → Request Payload:
```json
{
  "steam_id": "76561198000000000",
  "asset_ids": ["demo_1", "demo_2"],
  "option_days": 14
}
```

### 4. Проверьте URL

Когда нажимаете "Продолжить", URL должен быть:
```
http://localhost:3000/cabinet/quote?items=demo_1,demo_2,demo_3
```

## Если не работает:

### Вариант 1: Используйте демо-данные

В `backend/services/steam_service.py` метод `_get_demo_inventory()` возвращает предметы с `assetid` = "demo_1", "demo_2" и т.д.

Убедитесь что frontend отправляет именно эти ID.

### Вариант 2: Упростите проверку

В `backend/main.py` закомментируйте проверку:
```python
# if not selected_items:
#     raise HTTPException(status_code=400, detail="Предметы не найдены")

# Если предметы не найдены, используем все
if not selected_items:
    selected_items = all_items[:5]  # Берем первые 5
```

## Быстрый тест:

1. Откройте http://localhost:3000/cabinet/inventory
2. Нажмите "Загрузить инвентарь"
3. Введите Steam ID: 76561198000000000
4. Выберите любой предмет
5. Нажмите "Продолжить"
6. Должна открыться страница с расчетом

## Если все равно не работает:

Напишите мне что показывает консоль backend когда вы нажимаете "Продолжить".
